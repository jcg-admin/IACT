# Checklist de trazabilidad de requisitos

- [ ] Cada requisito funcional enlaza con al menos un caso de uso.
- [ ] Cada caso de uso enlaza con reglas de negocio correspondientes.
- [ ] Requisitos no funcionales referencian atributos de calidad y métricas.
- [ ] Matriz vertical actualizada (Business Case → Pruebas).
- [ ] Matriz horizontal documentada (conflictos y dependencias).
- [ ] Historial de cambios actualizado con versiones.
