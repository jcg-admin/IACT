# Espacio de documentación - Frontend

Este espacio agrupa la documentación del frontend y la organiza en los mismos dominios clave que el backend para mantener la paridad operativa:

- `arquitectura`: lineamientos de componentes, flujos de navegación y decisiones UX/UI.
- `checklists`: listas de verificación para entregables, revisiones visuales y accesibilidad.
- `devops`: procesos de build, empaquetado y despliegue del cliente.
- `diseno_detallado`: especificaciones de vistas, estados y contratos con el backend.
- `planificacion_y_releases`: roadmap, versiones y entregas progresivas.
- `qa`: estrategias de pruebas unitarias, funcionales y de accesibilidad.
- `requisitos`: historias de usuario, criterios de aceptación y guías de contenido.
- `gobernanza`: estándares de código, guías de estilo y políticas de contribución.

Cada carpeta cuenta con un README inicial para desarrollar la información correspondiente.
