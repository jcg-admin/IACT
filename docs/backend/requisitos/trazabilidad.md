# Trazabilidad de requisitos del backend

Utiliza la matriz maestra localizada en [`../../requisitos/trazabilidad.md`](../../requisitos/trazabilidad.md) para mantener la relación RQ ↔ UC ↔ TC.

Cuando se requieran vistas específicas del backend, añade secciones con:

1. **Servicios o módulos involucrados.** Referencias a clases o endpoints.
2. **Casos de uso internos.** Cómo se conectan con flujos compartidos.
3. **Cobertura de pruebas.** Enlaces a suites automatizadas o runbooks.

Mantén sincronizadas estas extensiones con los reportes de QA y las bitácoras de despliegue.
