# Requerimientos Funcionales (Nivel 4)

Marco conceptual: `docs/gobernanza/marco_integrado/marco_reglas_negocio.md`

## Descripción

Los requerimientos funcionales especifican las funcionalidades concretas que el backend debe proporcionar.

## Nomenclatura

`RF-NNN-{descripcion_breve}.md` o `rf{nnn}_{descripcion}.md`

Ejemplos:
- `RF-001-evaluacion-permisos.md`
- `rf001_evaluacion_permisos_tres_niveles.md`

## Estructura de Documento

- Identificador único
- Descripción
- Criterios de aceptación
- Prioridad
- Dependencias
- Trazabilidad a casos de uso (Nivel 3)

## Trazabilidad

Nivel 4 deriva de: Nivel 3 (Casos de Uso)
Nivel 4 se complementa con: Nivel 5 (Atributos de Calidad)

Ver: `../trazabilidad.md`
