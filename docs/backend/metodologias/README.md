# Metodologías - Backend

Metodologías y prácticas de desarrollo para el backend.

---

## Metodologías Disponibles

### [Test-Driven Development (TDD)](./TDD-metodologia.md)
Desarrollo guiado por tests: escribir tests antes del código.

**Cuándo usar:**
- Nuevo desarrollo de features
- Código crítico de negocio
- Refactoring de código legacy

**Beneficios:**
- Mayor confianza en el código
- Mejor diseño
- Documentación viva

### [Clean Architecture](./clean-architecture.md)
Principios de arquitectura limpia para código mantenible.

**Cuándo usar:**
- Diseño de nuevos módulos
- Refactoring de código existente
- Separación de concerns

**Beneficios:**
- Código testeable
- Independencia de frameworks
- Fácil mantenimiento

---

## Prácticas Recomendadas

1. **TDD para nuevo código**
2. **Clean Architecture para módulos grandes**
3. **Code reviews obligatorios**
4. **80%+ cobertura de tests**
5. **Documentación inline (docstrings)**
